- 对这段代码进行了理解：
`example : ∀ m n : Nat, Even n → Even (m * n) := fun m n ⟨k, (hk : n = k + k)⟩ ↦`
  `have hmn : m * n = m * k + m * k := by rw [hk, mul_add]`
  `show ∃ l, m * n = l + l from ⟨_, hmn⟩`

（以下理解参考Gemini，之后如果遇到不一致的理解还需要确认）
1. 命题即函数。这里的命题，就是要构造一个接受【自然数m】，【自然数n】，【n为偶数的证据】、并且输出【`m*n`是偶数的证据】的函数
2. 理解 `⟨k, (hk : n = k + k)⟩` 首先要理解 Even。Even n实际上就是∃ k, n = k + k，所以`⟨k, (hk : n = k + k)⟩` 会把Even n进行拆解，提取出数值k与命题hk。
	1. Remark：注意是`⟨`不是`<`，输入方法为`\<`
3. 理解`⟨_, hmn⟩`的时候需要理解：`_`是占位符，Lean可以自动推导，不需要显式写出`m*k`。`from`用于连接`show[类型]`和`[值]`。

- 对简化代码进行了理解：
`example : ∀ m n : Nat, Even n → Even (m * n) := by`
	`rintro m n ⟨k, hk⟩; use m * k; rw [hk]; ring`
- 理解了intro, rcases和rintro.
	- intro 用于 p \r q: 得到hp: p与目标q
	- intro 用于 \forall n: Nat, n+0=n: 得到一个变量n: Nat
	- rcases用于解构，可以拆解“存在量词”（`∃`）、“逻辑与”（`∧`）或“逻辑或”（`∨`）
	- rintro是两者的结合。本例中，如果不使用rintro，需要先用intro提取对象Even n，再进行rcases解构。
